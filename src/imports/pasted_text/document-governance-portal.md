社内ポータルに設置する「規程・文書管理ハブ」のUIをデザインしてください。

## 目的

会社では、規程・標準・プロセス定義・RACI・マニュアル・Playbook・Agent Instruction・証跡類が、各部署のSharePointサイトに分散して保管されています。

作業者が必要な最新版文書に迷わず到達でき、文書管理者が棚卸し・版管理・承認状況を確認でき、監査者が証跡・改定履歴・所管部署・承認ログを確認できるようにしたいです。

全社員向けポータルに置く画面ですが、表示モードを切り替えることで、作業者・管理者・監査者それぞれに最適化された表示にしてください。

---

規程・文書管理ハブ
Document Governance Portal

# 画面レイアウト

画面上部に大きくタイトルを表示してください。

サブコピー：

右上に以下の補助リンクを配置してください。

* 文書登録申請
* 権限申請
* 問い合わせ

## 2. 自然言語検索バー

ヘッダー直下に、横幅の広い自然言語検索バーを配置してください。

検索バーのプレースホルダー：
「何を確認したいですか？ 例：経費精算の最新版マニュアルを探して」

検索バーの下に、検索例をチップ形式で表示してください。

検索例チップ：

* 経費精算の最新版マニュアルを探して
* 情報セキュリティ規程の改定履歴を見たい
* 承認待ちの文書を一覧で出して
* 入退社プロセスに関係するRACIとPlaybookを見たい
* 権限申請が必要な文書を確認したい

検索バー右側に「検索」ボタンを配置してください。

## 3. 表示モード切替

自然言語検索バーの下に、表示モード切替タブを配置してください。

タブ：

* 作業者モード
* 管理者モード
* 監査者モード

初期表示は「作業者モード」にしてください。

各モードの説明文をタブ下に1行で表示してください。

作業者モード：
「必要な最新版文書をすばやく探すための表示です。」

管理者モード：
「文書の棚卸し、版管理、承認状況、レビュー期限を確認するための表示です。」

監査者モード：
「文書階層、改定履歴、承認ログ、関連証跡を確認するための表示です。」

## 4. メインタイルメニュー

表示モード切替の下に、タイル型メニューを配置してください。

タイルは4つです。

### タイル1：業務プロセスから探す

説明文：
「入退社、経費精算、インシデント対応、購買、契約など、業務の流れから関連文書を探します。」

### タイル2：文書種別から探す

説明文：
「規程、標準、手順書、マニュアル、様式、Playbookなど、文書の種類から探します。」

### タイル3：所管部署から探す

説明文：
「人事、経理、総務、情報システム、法務など、文書を管理する部署から探します。」

### タイル4：文書階層から探す

説明文：
「ポリシーから証跡まで、文書の上下関係をたどって探します。」

タイルはSharePointポータルに馴染む、シンプルで業務向けのデザインにしてください。
アイコンは控えめに使い、視認性を重視してください。

## 5. 文書階層ナビゲーション

メインタイルの下、または右側に、文書階層をたどれるナビゲーションを配置してください。

文書階層は以下です。

1. ポリシー
2. スタンダード / 規程類
3. プロセス定義
4. RACI
5. マニュアル / Playbook
6. Agent Instruction
7. 実行ログ / 判断ログ / 証跡

各階層を縦方向のステップ、または横方向のステップナビゲーションとして表現してください。

文書階層の見せ方は、監査者モードでは特に目立つようにしてください。
作業者モードでは、必要に応じて展開できる補助情報として扱ってください。

## 6. 状態別ショートカット

画面中央に、状態別ショートカットをカードまたはチップで表示してください。

ショートカット：

* 最新版
* よく使う文書
* 最近改定された文書
* 承認待ち・改定中
* 権限申請が必要な文書

それぞれに件数バッジを表示できるようにしてください。

例：

* 最新版：128件
* よく使う文書：24件
* 最近改定された文書：12件
* 承認待ち・改定中：8件
* 権限申請が必要：5件

管理者モードでは「承認待ち・改定中」を強調してください。
監査者モードでは「最近改定された文書」「証跡あり」を強調してください。
作業者モードでは「最新版」「よく使う文書」を強調してください。

# スタイル指示

Microsoft 365 / SharePoint の社内ポータルに馴染む、業務向けで信頼感のあるUIにしてください。

## トーン

* シンプル
* 業務向け
* 迷わない
* 監査に耐える
* 管理しやすい
* 派手すぎない
* SharePointポータルに自然に溶け込む

## 色

* ベース：白または薄いグレー
* メインカラー：Microsoft 365に馴染むブルー系
* 注意状態：オレンジまたはアンバー系
* エラー・期限切れ：赤系
* 完了・最新版：グリーン系
* 権限申請：パープルまたはグレー系

色だけで状態を表現せず、必ずラベル文字を併用してください。

## レイアウト

* 余白を広めに取り、詰め込みすぎない
* 全社員が使うため、専門用語を出しすぎない
* 管理者・監査者向けの詳細情報は、モード切替や展開表示で出す
* 作業者モードでは「最新版を開く」ボタンを最も目立たせる
* 管理者モードではステータス管理と一覧性を重視する
* 監査者モードでは文書階層と証跡への接続を重視する
